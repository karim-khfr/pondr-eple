# CONTRIBUTING.md

> **Guide de contribution -- Pondr**\
> Version 1.3.0 --- © 2026 Karim Khenifer

Merci de votre intérêt pour Pondr.

## Objectif

Les contributions doivent améliorer le logiciel sans remettre en cause
ses principes fondamentaux :

-   objectivité ;
-   transparence ;
-   reproductibilité ;
-   confidentialité des données.

## Types de contributions

Les contributions suivantes sont les bienvenues :

-   correction de bugs ;
-   amélioration des performances ;
-   amélioration de l'accessibilité ;
-   amélioration de la documentation ;
-   optimisation du code ;
-   nouvelles fonctionnalités compatibles avec l'architecture existante.

## Bonnes pratiques

Avant toute proposition :

1.  Vérifier qu'elle répond à un besoin réel.
2.  Conserver une architecture modulaire.
3.  Documenter les changements.
4.  Éviter les régressions fonctionnelles.

## Style de code

-   JavaScript moderne (ES6+)
-   Fonctions courtes et documentées
-   Nommage explicite
-   Séparation stricte des responsabilités
-   Commentaires réservés aux éléments non évidents

## Tests

Avant de proposer une modification, vérifier :

-   import Excel ;
-   import CSV ;
-   validation des données ;
-   calcul des scores ;
-   exports ;
-   recherche et filtres.

## Documentation

Toute évolution fonctionnelle doit être accompagnée d'une mise à jour de
la documentation concernée.

## Philosophie du projet

Pondr privilégie la simplicité, la robustesse et la lisibilité plutôt
que la multiplication des dépendances ou des frameworks.

Les contributions doivent respecter cette orientation.

## Signalement d'un problème

Lors d'un rapport de bug, fournir si possible :

-   la version de Pondr ;
-   le navigateur utilisé ;
-   le système d'exploitation ;
-   les étapes de reproduction ;
-   le comportement observé ;
-   le comportement attendu.

Merci de contribuer à l'amélioration de Pondr.
